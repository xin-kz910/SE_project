# 📁 專案結構說明

## 🟦 1. Python 主程式

| 檔名         | 說明                                         |
| ---------- | ------------------------------------------ |
| **app.py** | 整個網站的主程式，負責路由（Routing）、處理前端回傳資料、呼叫資料庫功能。   |
| **db.py**  | 資料庫連線與 CRUD 函式（新增/修改/查詢/刪除）。所有資料庫操作都由這裡完成。 |

---

## 🟦 2. Templates（前端 HTML）

| 檔案                      | 功能                                            |
| ----------------------- | --------------------------------------------- |
| **base.html**           | 網站所有頁面的共同框架（Navbar / layout）。其他頁面都會 extend 它。 |
| **login.html**          | 使用者登入頁面。                                      |
| **register.html**       | 使用者註冊頁面。                                      |
| **projects_list.html**  | 顯示所有案件的列表頁面。                                  |
| **project_detail.html** | 單一案件的詳細資訊（內容、招標、交付資訊）。                        |
| **project_create.html** | 新增案件的頁面。                                      |
| **project_edit.html**   | 編輯案件的頁面。                                      |

---

## 🟦 3. static/（放 CSS、JS、圖片）

| 檔案            | 功能            |
| ------------- | ------------- |
| **style.css** | 網站主要的版面與樣式設定。 |
| **upload 資料夾** | 放結案檔案的地方。 |

---

## 🟦 4. docs/（文件資料夾，若有）

| 檔案             | 功能                   |
| -------------- | -------------------- |
| **schema.sql** | 專案資料庫結構與資料。 |

---

# 🗄 資料庫結構（簡述）

資料庫包含以下主要資料表：

| 資料表            | 說明                            |
| -------------- | ----------------------------- |
| **users**      | 儲存使用者資訊（帳號/密碼/Email）。         |
| **projects**   | 專案資料（標題/內容/建立者/得標者等）。         |
| **bids**       | 來自接案者的投標紀錄（專案 ID、使用者 ID、報價等）。 |
| **deliveries** | 最後交付檔案資訊（上傳的檔案、文字說明、交付者）。     |

---

# 🚀 如何啟動專案（本地端）

```bash
fastapi dev app.py
```

伺服器會啟動在：

```
http://127.0.0.1:8000/
```
